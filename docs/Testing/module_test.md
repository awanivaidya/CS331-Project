# Module Testing Report - Risk Assessment and Alerts Module

## Test Plan

### 1. Objective of Testing
The objective is to verify that the Risk Assessment and Alerts module correctly:
- classifies customer risk status from sentiment scores,
- updates rolling sentiment history and weighted average sentiment,
- generates alerts for warning/critical customers,
- avoids incorrect behavior for invalid edge-case inputs.

## Defect Analysis (Found During Exploratory Testing)

### Bug ID: BUG-RISK-001
- Description of the issue: `generateAlertIfNeeded` crashes when `project` is null/undefined for warning/critical flows.
- Steps to reproduce:
  1. Import `generateAlertIfNeeded` from `riskService.js`.
  2. Call with critical customer and `project = null`.
- Expected vs Actual Result:
  - Expected: Function should safely handle missing project and either return null or fallback alert with safe defaults.
  - Actual: Runtime error `Cannot read properties of null (reading '_id')`.
- Severity level: High
- Suggested fix:
  - Add guard before accessing `project._id` and `project.name`.
  - Use optional chaining/defaults: `project?._id || null`, `project?.name || "Unknown Project"`.

- Reproduction code:
  node -e "const { generateAlertIfNeeded } = require('./core/services/riskService'); const customer = { _id: 'cust-1', name: 'ACME Corp', sentimentScore: -0.82, riskStatus: 'critical' }; console.log(generateAlertIfNeeded(customer, null));"

### Bug ID: BUG-RISK-002
- Description of the issue: `updateCustomerSentiment` crashes if `customer.sentimentHistory` is missing.
- Steps to reproduce:
  1. Import `updateCustomerSentiment`.
  2. Pass customer object without `sentimentHistory`.
  3. Call function with a valid score.
- Expected vs Actual Result:
  - Expected: Function should initialize missing history to empty array and continue.
  - Actual: Runtime error `Cannot read properties of undefined (reading 'push')`.
- Severity level: High
- Suggested fix:
  - Initialize history safely at function start:
    - `if (!Array.isArray(customer.sentimentHistory)) customer.sentimentHistory = [];`
- Reproduction code:
node -e "const { updateCustomerSentiment } = require('./core/services/riskService'); const customer = { sentimentScore: 0, riskStatus: 'stable' }; updateCustomerSentiment(customer, -0.5); console.log(customer);"

### Bug ID: BUG-RISK-003
- Description of the issue: Non-numeric `newScore` causes `NaN` sentiment score and incorrect `stable` risk classification.
- Steps to reproduce:
  1. Import `updateCustomerSentiment`.
  2. Pass valid customer object with history.
  3. Call with `newScore = "bad-input"`.
- Expected vs Actual Result:
  - Expected: Input should be rejected (validation error) or safely converted and validated.
  - Actual: `sentimentScore` becomes `NaN`; `riskStatus` resolves to `stable`, causing silent logical corruption.
- Severity level: Medium
- Suggested fix:
  - Validate numeric inputs before calculations:
    - `const score = Number(newScore); if (!Number.isFinite(score)) throw new Error("Invalid sentiment score");`
- Reproduction code:
node -e "const { updateCustomerSentiment } = require('./core/services/riskService'); const customer = { sentimentHistory: [{ score: 0, timestamp: new Date() }], sentimentScore: 0, riskStatus: 'stable' }; updateCustomerSentiment(customer, 'bad-input'); console.log(customer);"
